Key Principles Learned
  • ›Always create TrueNAS datasets via the UI before running bash setup scripts — prevents ACL conflicts
  • ›ACL permission issues on TrueNAS: use sudo su rather than standard chown/chmod
  • ›NordVPN city restrictions in Gluetun cause auth failures — remove SERVER_CITIES for auto best-server
  • ›Cloudflare DNS resolution inside Docker requires explicit DNS servers (1.1.1.1, 8.8.8.8)
  • ›TRaSH quality profiles need manual tuning after sync — YTS flagged as LQ, size minimums can block releases
  • ›Maintain hardlink-compatible folder structure at all times for efficient media management

Folder Structure (Hardlink-Compatible)
The hardlink layout ensures qBittorrent seeds while Sonarr/Radarr manage files — no duplicate storage.
# Hardlink-compatible media structure on tank pool /mnt/tank/data/ ├── media/ │ ├── movies/ # Radarr managed │ └── tv/ # Sonarr managed └── torrents/ ├── movies/ # qBittorrent downloads └── tv/ # Config storage on SSD pool /mnt/storage/configs/ ├── jellyfin/ ├── sonarr/ ├── radarr/ ├── prowlarr/ ├── qbittorrent/ ├── immich/ ├── romm/ ├── code-server/ └── uptime-kuma/
💡 Why Hardlinks Matter
When qBittorrent finishes a download, Sonarr/Radarr create a hardlink instead of copying the file. The "copy" in /media/ and the original in /torrents/ share the same disk blocks — two directory entries for the price of one file. Seeding continues while your library stays organized.

How Cloudflare Tunnels Work
A Cloudflare Tunnel (via the cloudflared daemon in Docker) creates an outbound-only encrypted connection from your server to Cloudflare's edge. Internet traffic hits Cloudflare first, then routes through the tunnel to your container — your home IP is never exposed and no router ports need to be opened.
# cloudflared container in arrstack compose cloudflared: image: cloudflare/cloudflared:latest restart: unless-stopped command: tunnel run environment: - TUNNEL_TOKEN=your_token_here
Docker DNS Fix for Cloudflare Domains
⚠️ Important
By default, Docker containers use the host's DNS resolver, which may not resolve Cloudflare-managed custom domains internally. Always add explicit DNS servers to compose stacks that need to reach other services by domain name.
# Add to any service that needs to resolve gonzotech.us internally services: your-service: dns: - 1.1.1.1 - 8.8.8.8

Services
🔒 Gluetun
VPN GATEWAY CONTAINER
All download traffic routes through Gluetun using NordVPN OpenVPN. qBittorrent uses Gluetun's network namespace — it can only reach the internet through the VPN tunnel.
VPN_SERVICE_PROVIDER=nordvpn VPN_TYPE=openvpn # SERVER_CITIES removed — causes auth failures # Auto best-server selection is more reliable
⬇️ qBittorrent
DOWNLOAD CLIENT
Runs on Gluetun's network. All traffic is VPN-protected. Downloads land in /torrents/ and Sonarr/Radarr hardlink to /media/.
network_mode: service:gluetun
🔍 Prowlarr
INDEXER MANAGER
Central indexer hub that syncs trackers to both Sonarr and Radarr. Manages all tracker credentials in one place.
🕷️ FlareSolverr
CLOUDFLARE BYPASS
Solves Cloudflare challenges for indexers that use bot protection. Prowlarr routes through FlareSolverr when needed to access protected tracker sites.
📡 Sonarr
TV SERIES MANAGER · TRaSH: WEB-1080p
Monitors and automatically downloads TV shows. Quality profile synced from TRaSH Guides via Recyclarr.
🎬 Radarr
MOVIE MANAGER · TRaSH: HD Bluray + WEB
Monitors and automatically downloads movies. Quality profile synced from TRaSH Guides via Recyclarr.
Recyclarr (TRaSH Guides Sync)
♻️ Recyclarr
Automatically syncs TRaSH Guide quality profiles and custom formats to Sonarr and Radarr. Keeps quality settings current without manual maintenance.
# Key tuning applied after initial sync: # - Minimum custom format score: -10000 (broader compatibility) # - Quality size minimums lowered (prevents blocking smaller releases) # - YTS releases: acceptable despite LQ flag for certain content
⚠️ Post-Sync Required
Always review quality profile settings after a Recyclarr sync. Default TRaSH profiles can block common releases with oversized minimum file size thresholds and strict custom format scoring.

Dataset Guide
// TRUENAS BEST PRACTICES FOR NEW APPS
● Reference
📌 Golden Rule
Always create datasets in the TrueNAS UI FIRST, before running any bash scripts to create subfolders or set permissions. Creating datasets via bash can conflict with ACLs set by TrueNAS and cause permission errors.
Step 1 — Create Dataset via TrueNAS UI
  1. Go to Storage in the TrueNAS UI sidebar
  2. Click the ⋮ (three dots) next to the pool you want → Add Dataset
  3. Set Name — e.g. appname
  4. Set Dataset Preset → Apps
  5. Leave ACL Type as POSIX
  6. Leave Compression as lz4
  7. Click Save
Step 2 — Bash Setup (After Dataset Created)
# Run as root to avoid ACL conflicts sudo su # Create subdirectory structure mkdir -p /mnt/storage/configs/myapp # Set ownership to match container PUID/PGID chown -R 1000:1000 /mnt/storage/configs/myapp # Set permissions chmod -R 755 /mnt/storage/configs/myapp # Verify ls -la /mnt/storage/configs/
Dataset Settings Cheat Sheet
Compressionlz4 — Default. Fast + decent ratio, good for most uses.
ACL TypePOSIX — Best for Docker containers (matches Linux permissions model)
Case SensitivitySensitive — Keep default unless Windows SMB shares need otherwise
QuotaOptional — Set on media datasets to prevent runaway disk usage
SyncStandard — Default. "Always" is safer but slower for large writes
Snapshot TaskConfigure periodic snapshots on config datasets for easy rollback
