Folder Structure (Hardlink-Compatible)
The hardlink layout ensures qBittorrent seeds while Sonarr/Radarr manage files — no duplicate storage.
# Hardlink-compatible media structure on tank pool /mnt/tank/data/ ├── media/ │ ├── movies/ # Radarr managed │ └── tv/ # Sonarr managed └── torrents/ ├── movies/ # qBittorrent downloads └── tv/ # Config storage on SSD pool /mnt/storage/configs/ ├── jellyfin/ ├── sonarr/ ├── radarr/ ├── prowlarr/ ├── qbittorrent/ ├── immich/ ├── romm/ ├── code-server/ └── uptime-kuma/
💡 Why Hardlinks Matter
When qBittorrent finishes a download, Sonarr/Radarr create a hardlink instead of copying the file. The "copy" in /media/ and the original in /torrents/ share the same disk blocks — you get two directory entries for the price of one file. Seeding continues while your library is organized.

How Cloudflare Tunnels Work
A Cloudflare Tunnel creates an outbound-only encrypted connection from the homelab server to Cloudflare's edge. Internet traffic hits Cloudflare first, then routes through the tunnel — the home IP is never exposed and no firewall ports need to be opened.
Docker DNS Fix for Cloudflare Domains
⚠️ Important
By default, Docker containers use the host's DNS resolver, which may not resolve Cloudflare-managed custom domains internally. Always add explicit DNS servers to compose stacks that need to reach other services by domain name.
# Add to any service that needs to resolve gonzotech.us internally services: your-service: dns: - 1.1.1.1 - 8.8.8.8

Pipeline Overview
Seerr → Sonarr/Radarr → Prowlarr → qBittorrent → through Gluetun VPN tunnel → Download → Hardlink to /media/ → Jellyfin picks up automatically
Services
🔒 VPN Gateway
ALL DOWNLOAD TRAFFIC VPN-PROTECTED
All download traffic routes through a VPN gateway container. qBittorrent and other download services use the VPN's network namespace, meaning they can only access the internet through the encrypted tunnel.
network_mode: service:vpn
⬇️ qBittorrent
DOWNLOAD CLIENT
Runs on Gluetun's network. All traffic is VPN-protected. Configured with the hardlink-compatible folder structure so downloads land in /torrents/ and Sonarr/Radarr hardlink to /media/.
network_mode: service:gluetun
🔍 Prowlarr
INDEXER MANAGER
Central indexer hub that syncs trackers to both Sonarr and Radarr. Manages all tracker credentials in one place rather than duplicating across arr apps.
🕷️ FlareSolverr
CLOUDFLARE BYPASS
Solves Cloudflare challenges for indexers that use bot protection. Prowlarr routes requests through FlareSolverr when needed to access protected tracker sites.
📡 Sonarr
TV SERIES MANAGER · TRaSH: WEB-1080p
Monitors and automatically downloads TV shows. Quality profile synced from TRaSH Guides via Recyclarr.
🎬 Radarr
MOVIE MANAGER · TRaSH: HD Bluray + WEB
Monitors and automatically downloads movies. Quality profile synced from TRaSH Guides via Recyclarr.
Recyclarr (TRaSH Guides Sync)
♻️ Recyclarr
Automatically syncs TRaSH Guide quality profiles and custom formats to Sonarr and Radarr. Keeps quality settings current without manual maintenance.
# Key tuning applied after initial sync: # - Minimum custom format score: -10000 (broader compatibility) # - Quality size minimums lowered (prevents blocking smaller releases) # - YTS releases: acceptable despite LQ flag for certain content
⚠️ Post-Sync Required
Always review quality profile settings after a Recyclarr sync. Default TRaSH profiles can block common releases with oversized minimum file size thresholds and strict custom format scoring.

Dataset Guide
// TRUENAS BEST PRACTICES FOR NEW APPS
● Reference
📌 Golden Rule
Always create datasets in the Storage UI FIRST, before running any bash scripts to create subfolders or set permissions. Creating datasets via bash can conflict with existing ACLs and cause permission errors.
Step 1 — Create Dataset via TrueNAS UI
  1. Go to Storage in the management UI sidebar
  2. Click the ⋮ (three dots) next to the pool you want (tank or storage) → Add Dataset
  3. Set Name — e.g. appname
  4. Set Dataset Preset → Apps
  5. Leave ACL Type as POSIX
  6. Leave Compression as lz4
  7. Click Save
Step 2 — Bash Setup (After Dataset Created)
# Run as root to avoid ACL conflicts sudo su # Create subdirectory structure mkdir -p /mnt/storage/configs/myapp # Set ownership to match container PUID/PGID chown -R 1000:1000 /mnt/storage/configs/myapp # Set permissions chmod -R 755 /mnt/storage/configs/myapp # Verify ls -la /mnt/storage/configs/
Dataset Settings Cheat Sheet
Compressionlz4 — Default, great for most uses. Fast + decent ratio.
ACL TypePOSIX — Best for Docker containers (matches Linux permissions model)
Case SensitivitySensitive — Keep default unless Windows SMB shares need it
QuotaOptional — Set on media datasets to prevent runaway disk usage
SyncStandard — Default. "Always" is safer but slower for large writes
Snapshot TaskConfigure periodic snapshots on config datasets for easy rollback

Authentik SSO
// IDENTITY PROVIDER · auth.gonzotech.us
◐ In Progress
🛡️ What It Does
Authentik is the central identity layer for GonzoTech. All public services authenticate through Authentik — one login, one place to manage users, MFA enforcement, audit logs, and instant access revocation across all services simultaneously. Google is configured as a social login source so you sign in with your existing Google account.
Why Authentik
🔑 Single Login
One set of credentials — or just your Google account — gets you into Jellyfin, Immich, Seerr, and any other connected service. No separate passwords to manage per app.
🛑 Instant Revocation
Removing a user in Authentik immediately cuts off access to every connected service. No hunting through individual app settings.
📋 Audit Trail
Every login attempt — successful or failed — is logged centrally. Full visibility into who accessed what and when.
🔐 MFA Everywhere
Configure MFA once in Authentik and it applies to all connected services automatically. No per-app MFA setup required.
